# Satellite Change Detection - Mobile Client (SatelliteApp)

The **SatelliteApp** is a production-ready mobile application built with **React Native** and **Expo**. It serves as the primary user interface for the Satellite Change Detection system, allowing farmers, agronomists, and administrators to monitor agricultural fields, view satellite-derived change maps, and manage real-time alerts.

By communicating directly with the backend microservices via the **API Gateway**, it provides a seamless, cross-platform (iOS and Android) experience backed by a robust and highly scalable event-driven architecture.

---

## 🚀 Key Features

- **Field Monitoring Dashboard (`index.tsx`):** A centralized view of all registered agricultural fields, displaying their status and quick metrics.
- **Change Maps (`change-map.tsx`):** Visual representations of NDVI (Normalized Difference Vegetation Index) passes and the calculated temporal differences between satellite imagery, helping to pinpoint crop stress or growth.
- **Alert Management (`alerts.tsx`):** Real-time display of critical and positive agricultural notifications generated by the backend classification and alert services. Includes functionality to acknowledge/dismiss alerts.
- **Modern Styling:** Fully styled using **NativeWind** (Tailwind CSS for React Native), ensuring a responsive, accessible, and premium UI.
- **File-Based Routing:** Utilizes **Expo Router** for intuitive and robust navigation across the app.

---

## 🛠 Tech Stack

- **Framework:** React Native 0.85 & React 19
- **Build Tool:** Expo SDK 56
- **Routing:** Expo Router v56
- **Styling:** NativeWind (Tailwind CSS v3)
- **Language:** TypeScript

## ⚙️ Setup & Installation

### Prerequisites

- **Node.js** (v18+ recommended)
- **npm** or **yarn**
- **Expo Go** app installed on your physical mobile device (iOS/Android), OR
- Android Studio Emulator / iOS Simulator configured on your machine.
- The backend **API Gateway** must be running (typically on `http://localhost:8080`) to provide data to the app.

### Installation

1. **Navigate to the app directory:**
   ```bash
   cd SatelliteApp
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

### Configuration

Ensure the application knows where to find the backend API Gateway. If you are running the backend locally and testing on a physical device, you may need to update the API base URL in the service configuration (e.g., `src/constants` or `src/service`) to point to your machine's local IP address (e.g., `http://192.168.1.x:8080/api`) instead of `localhost`.

### Running the Application

Start the Expo development server:

```bash
npm start
```
*(Or use `npm run android` / `npm run ios` to bypass the selection menu).*

Once the Metro bundler starts, you can:
- **Scan the QR Code** with the Expo Go app on your physical device.
- Press `a` to open in the Android Emulator.
- Press `i` to open in the iOS Simulator.

## 📁 Project Structure

The codebase is organized under the `src/` directory to cleanly separate application code from configuration:

- **`src/app/`**: Contains the file-based routing screens (`index.tsx`, `alerts.tsx`, `change-map.tsx`) and the global layout (`_layout.tsx`).
- **`src/components/`**: Reusable UI components.
- **`src/service/`**: API integration layer for communicating with the backend API Gateway (`/api/data`, `/api/alerts`, etc.).
- **`src/constants/`**: Application-wide constants, theme configurations, and API URL definitions.
- **`src/types.ts`**: Global TypeScript interfaces modeling the backend responses (Fields, Alerts, Change Records).
- **`src/global.css`**: Global stylesheet initializing NativeWind and Tailwind utility classes.

---

## 🔄 Integration Context

The mobile app is a pure frontend client. It maintains no local persistent database and relies entirely on the **API Gateway** for its state. 
- It fetches field data from the `data-service`.
- It fetches and acknowledges alerts from the `alert-service`.
- It visualizes change data generated by the `change-detection-service` and `classification-service`.
